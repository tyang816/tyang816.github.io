4.3 模型验证【斯坦福21秋季：实用机器学习】

2022-06-06 斯坦福21秋季：实用机器学习 0 0 评论 字数统计: 556(字) 阅读时长: 1(分)

Estimate Generalization Error

  • 在测试数据集上近似得到泛化误差，这个数据集只能被使用一次
    • 比如就是期中考试，就考一次不能考第二次
    • 房价模型预测与真实价格的区别
    • Kaggle私榜结果
  • 验证数据集：可以被用多次
    • 通常是训练数据的一部分
    • 当使用作为“测试”的时候，大部分情况下指的是“验证”

Hold Out Validation

  • 把数据集分割成训练和验证集
    • 在训练集上训练，在验证集上近似误差
  • 通常随机选择 n% 样本作为训练集
    • 通常的选择为 n=50,40,30,20,10

Split non I.I.D. data

  • 随机分割数据不一定起作用
  • 时序信息：比如房价，股票价格
    • 验证集通常在训练集之后
  • 样本属于不同的组：比如同一个人的不同照片
    • 根据组分割而不是样本
  • 不平衡数据
    • 从小的类里面更多采样，否则由于随机采样更多采的是大的类别里面的数据，导致结果偏高

K-fold Cross Validation

  • 数据不足够时候有用
  • 算法
    • 将训练数据切成K片
    • For i =1,…,K
      • 用一个第i块作为验证集，剩下的作为训练集
    • 汇报K个验证集的平均损失
  • 常用的选择：K=5或10

Common Mistakes

  • 发现结果特别好90%可能是bug
    • 有可能是验证集被污染了
  • 验证集的样本来源于训练集
    • 重复的样本
    • 数据融合问题，多个数据源的数据可能有冗余性
      • 比如从搜索引擎上搜图片来评估ImageNet上的结果，看似很公平，但是ImageNet本身就是从搜索引擎上获取的，所以数据可能会有一定的重合
  • 数据泄露
    • 使用了未来数据来预测

Summary

  • 评估模型时用测试数据来评估
  • 从训练集采样出验证集近似得到泛化误差
    • 可以选择模型评估很多次
    • 验证集要和测试集尽量相似，尽量拟合实际生产环境
    • 错误的验证集是一个常见的错误
